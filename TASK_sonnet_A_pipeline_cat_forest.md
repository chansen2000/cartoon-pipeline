### 1.4 森林背景资产烘焙 — 新建 `tools/gen_bg_assets.sh`

**目标**: 把每图层 PNG → LVGL RGB565A8 C 数组(图层间有 alpha 透明度,必须保 alpha,**不要用纯 RGB565**)

**输入**:
- `output/{material}/json/{tw}_{th}/variant_*_layers.json` — 图层清单(决定要烘哪些)
- 原图: `素材/选择/{material}/_PNG/{variant}/layers/{layer}.png` 或 `素材/选择/{material}/PNG/{variant}/layers/{layer}.png`(自动探测,跟 BackgroundAssembler `_find_dir` 一致)

**输出目录**: `output/{material}/lvgl_export/bg/`
**输出文件**: `bg_{material}_{variant}_{layer}.c`(material 用拼音或保留中文都可,但变量名必须 ASCII — **建议用 BG_PINYIN_MAP** 给中文素材一个 ASCII 别名,例: `森林` → `forest`)

**变量名规则**(LVGL 端 LV_IMAGE_DECLARE 引用):
```
bg_{ascii_material}_{variant}_{layer_safe}
```
其中 `layer_safe` = layer 名替换非法字符 → `_`(已是 `l1_background` 这种格式,直接用)

**例**: `森林/variant_01/l1_background` → 变量名 `bg_forest_01_l1_background`

**位置参数**:
- `$1` = material(默认 `小猫`,但本脚本只对背景包有意义)
- 必须支持两种素材路径(`_PNG` 和 `PNG`)

**目标尺寸**: **410×502**(2026-05-28 sheng 拍板,跟角色同坐标系,内存 4.2 MB/变体)

体积估算:
- RGB565A8 = 3 byte/pixel
- 410 × 502 × 3 = 617 KB/图层
- 森林 7 图层 × 4 变体 = 17 MB 全量,单变体 4.2 MB
- ESP32-P4 32MB PSRAM 可承载

**实施要点**: 流水线已经在 `output/森林/json/1920_1080/` 出了 1920 视角的 JSON,需要重跑 `assemble_bg.py --material 森林 --all --target 410x502 --export-lvgl` 出 410 视角的 JSON 和 PNG,然后 `gen_bg_assets.sh` 读 `output/森林/json/410_502/variant_*_layers.json` 烘 410 尺寸的 RGB565A8 C 数组。

**禁止**: 不要保留 1920 视角的产物作为 LVGL 资产(只 410 进 LVGL,1920 留作未来视差升级用)。

**实现要点**:
```bash
# 1. 读 JSON 拿到本变体所有图层名
# 2. 找到原图路径(_PNG vs PNG 自动探测)
# 3. magick "$src" -resize {w}x{h} /tmp/_bg_resized.png
# 4. npx lv_img_conv -f -i "$var_name" -c CF_RGB565A8 -o "$out.c" /tmp/_bg_resized.png
# 5. sed 后处理 v8 → v9 (跟 gen_pngseq.sh 一样)
```

**找不到源图 → exit 1 报错**(跟 gen_part_dscs.sh 一致)

### 1.5 森林背景元数据 C 翻译 — 新建 `tools/gen_bg_meta.py`

**目标**: 把图层 JSON 翻译成 LVGL 可消费的 C 头/源,提供 `bg_variant_get(material, variant, &count)` 查表 API。

**输入**: `output/{material}/json/{tw}_{th}/variant_*_layers.json`
**输出**:
- `output/{material}/lvgl_export/meta/bg_{ascii_material}.h`
- `output/{material}/lvgl_export/meta/bg_{ascii_material}.c`

**.h 内容**:
```c
/* AUTO-GENERATED by 素材流水线/tools/gen_bg_meta.py. Do not edit. */
#ifndef BG_FOREST_H
#define BG_FOREST_H

#include "lvgl/lvgl.h"

typedef struct {
    const char            *name;            /* 图层名 (调试 + INI 索引) */
    int16_t                z_order;
    int16_t                parallax_ratio_x100;  /* 0..100, 实际值 = /100 */
    int16_t                w, h;
    const lv_image_dsc_t  *img;
} bg_layer_meta_t;

LV_IMAGE_DECLARE(bg_forest_01_l1_background);
LV_IMAGE_DECLARE(bg_forest_01_l2_trees01);
... /* 每变体每图层一行 */

extern const bg_layer_meta_t bg_forest_01[];
extern const uint8_t          bg_forest_01_count;
... /* 每变体一组 */

const bg_layer_meta_t *bg_forest_get(const char *variant, uint8_t *out_count);

#endif
```

**.c 内容**: 数组按 z_order 升序排好(跟 cat_parts_meta.c 同套路),`bg_forest_get` 是简单 strcmp 链。

**parallax_ratio 编码**: JSON 里是 `0.17` 这种 float,C 端用 `int16_t parallax_ratio_x100 = round(ratio * 100)`,LVGL 端除 100 还原。**避免 float**。

### 1.7 校验工具 — 新建 `tools/validate_animation_config.py`

**目标**: build_assets.sh 末段调用,任一不一致就 exit 1,不让坏数据流到 LVGL。

**校验项**:
1. **小猫 INI**:
   - 每个 `[action.X]` 的 X(idle/walk/...)在 `output/小猫/lvgl_export/pngseq/` 必须有对应文件
   - `[action.X].frame_count` 必须等于该 action 实际 pngseq 文件数(`ls pngseq_C01_X_*.c | wc -l`)
   - `[action.X].asset_prefix` 必须匹配实际文件名前缀
   - `[part.X]` 的 X 必须能映射到 `output/小猫/lvgl_export/meta/cat_parts_meta.h` 里的某个部件名(eye→Eye1, tail→Tails, hat→Hat 这种映射先硬编码 in tool)

2. **森林 INI**:
   - 每个 `[bg.{layer}]` 的 layer 必须存在于 `output/森林/json/1920_1080/variant_*_layers.json` 任一变体的 layers 数组中
   - `[bg.global]` 必须存在

任一失败 → 打印明细 → `sys.exit(1)`

### 1.8 编排 — 改 `build_assets.sh`

**目标流程**(P0 范围: 小猫 + 森林):

```bash
set -e
LVGL_PROJECT_DIR=...
LVGL_ASSETS_DIR=$LVGL_PROJECT_DIR/src/cartoon/assets
LVGL_BG_DIR=$LVGL_ASSETS_DIR/bg
LVGL_CONFIG_DIR=$LVGL_PROJECT_DIR/src/cartoon/config

# ── 角色: 小猫 ──
echo "[1] gen_lvgl_meta --material 小猫"
python3 tools/gen_lvgl_meta.py --material 小猫

echo "[2] gen_part_dscs 小猫"
bash tools/gen_part_dscs.sh 小猫

echo "[3] gen_pngseq 小猫 01"
bash gen_pngseq.sh 小猫 01

# ── 背景: 森林 ──
echo "[4] assemble_bg --material 森林 --all --target 410x502 --export-lvgl"
python3 assemble_bg.py --material 森林 --all --target 410x502 --export-lvgl

echo "[5] gen_bg_assets 森林"
bash tools/gen_bg_assets.sh 森林

echo "[6] gen_bg_meta 森林"
python3 tools/gen_bg_meta.py --material 森林

# ── 校验 ──
echo "[7] validate_animation_config"
python3 tools/validate_animation_config.py

# ── frame_lookup (LVGL 端工具,先 cp pngseq 再调) ──
echo "[8] gen_frame_lookup"
mkdir -p "$LVGL_ASSETS_DIR/pngseq"
cp -v output/小猫/lvgl_export/pngseq/pngseq_C01_*.c "$LVGL_ASSETS_DIR/pngseq/"
python3 "$LVGL_PROJECT_DIR/tools/gen_frame_lookup.py" \
    "$LVGL_ASSETS_DIR/pngseq" \
    "$LVGL_ASSETS_DIR/frame_lookup.c"

# ── 拷贝产物到 LVGL 工程 ──
echo "[9] cp assets → LVGL"
cp -v output/小猫/lvgl_export/meta/cat_parts_meta.{h,c} "$LVGL_ASSETS_DIR/"
cp -v output/小猫/lvgl_export/dsc/cat_C*.c "$LVGL_ASSETS_DIR/"

mkdir -p "$LVGL_BG_DIR/森林"
cp -v output/森林/lvgl_export/bg/bg_forest_*.c "$LVGL_BG_DIR/森林/"
cp -v output/森林/lvgl_export/meta/bg_forest.{h,c} "$LVGL_BG_DIR/森林/"

cp -v config/小猫/animation_config.ini "$LVGL_CONFIG_DIR/cat_animation_config.ini"
cp -v config/森林/animation_config.ini "$LVGL_CONFIG_DIR/bg_forest_animation_config.ini"

echo "Done."
```

**注意**:
- 小猫 INI 在 LVGL 端落地为 `cat_animation_config.ini`(给文件名加前缀,以免将来 6 角色 + 9 背景叠在同一目录冲突)
- 森林 INI 落地为 `bg_forest_animation_config.ini`
- LVGL 端 `cartoon_config_load()` 现在读固定文件名,改读多文件这一步**属于 kaifu 任务书 C**,不在本任务
